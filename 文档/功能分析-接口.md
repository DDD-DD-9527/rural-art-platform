# 乡村艺术平台 - 功能分析与接口文档

## 项目概述

本项目是一个乡村艺术学习与交流平台，包含用户管理、社交功能、帖子分享、评论互动、课程学习等核心模块。

#### 基础配置:
- **Base URL**: `import.meta.env.VITE_API_BASE_URL || '/api'`

## 数据模型分析

### 1. 用户模型 (User)
- **基础信息**: userId, username, email, password
- **个人资料**: profile (nickname, avatar, bio, location, gender)
- **学习统计**: learningStats (studyDays, completedCourses, totalPoints等)
- **社交统计**: socialStats (followersCount, followingCount)
- **偏好设置**: preferences (language, theme)
- **权限角色**: role (user, admin, creator)

### 2. 帖子模型 (Post)
- **基础信息**: title, content, type, author
- **媒体内容**: images, videos
- **分类标签**: tags, category
- **位置信息**: location (province, city, district, coordinates)
- **统计数据**: stats (viewCount, likeCount, commentCount, shareCount)
- **状态管理**: status (draft, published, hidden), visibility (public, private, followers)

### 3. 评论模型 (Comment)
- **基础信息**: content, author, targetType, targetId
- **层级结构**: parentComment (支持多级回复)
- **互动功能**: mentions (提及用户), stats (likeCount, replyCount)
- **状态管理**: status (published, hidden, deleted)

### 4. 关注模型 (Follow)
- **关系信息**: follower (关注者), following (被关注者)
- **时间记录**: createdAt (关注时间)

### 5. 点赞模型 (Like)
- **基础信息**: user, targetType, targetId, targetAuthor
- **时间记录**: createdAt (点赞时间)

## 接口特性分析

### 1. 认证与授权
- **JWT Token认证**: 使用Bearer Token进行身份验证
- **角色权限控制**: 区分普通用户、创作者、管理员权限
- **可选认证**: 部分接口支持游客访问

### 2. 数据验证
- **输入验证**: 使用express-validator进行参数验证
- **数据格式检查**: 严格的数据类型和长度限制
- **安全防护**: 防止SQL注入、XSS攻击

### 3. 分页与排序
- **统一分页**: page, limit参数标准化
- **多字段排序**: 支持按时间、热度、点赞数等排序
- **搜索过滤**: 支持关键词搜索和多条件筛选

### 4. 性能优化
- **数据预加载**: 使用populate预加载关联数据
- **统计缓存**: 热门数据和统计信息缓存
- **分页限制**: 合理的分页大小限制

### 5. 错误处理
- **统一错误格式**: 标准化的错误响应结构
- **详细错误信息**: 开发环境提供详细错误堆栈
- **用户友好提示**: 生产环境提供用户友好的错误信息

## 接口使用建议

### 1. 前端调用规范
- 统一使用API服务层，避免直接调用axios
- 合理处理loading状态和错误提示
- 实现请求去重和防抖机制

### 2. 数据缓存策略
- 用户信息和配置数据适合长期缓存
- 动态内容如帖子列表适合短期缓存
- 实时数据如点赞状态不建议缓存

### 3. 性能优化建议
- 合理设置分页大小，避免一次加载过多数据
- 使用虚拟滚动处理长列表
- 图片懒加载和压缩优化

### 4. 安全注意事项
- 敏感操作需要二次确认
- 文件上传需要类型和大小限制
- 用户输入内容需要XSS防护
