# 乡村艺术学习平台部署指南 (生产环境)

## 概述

本文档提供乡村艺术学习平台在生产环境中的完整部署指南，包括Docker容器化部署、Nginx反向代理、SSL证书配置、监控和备份策略。

## 系统架构

### 生产环境架构
[用户] → [Nginx反向代理:8080] → [前端容器:80] → [后端API:3000] → [MongoDB:27017]
↓
[静态文件服务]

### 技术栈
- **前端**: Vue.js 3 + Vite (Nginx容器)
- **后端**: Node.js + Express (Docker容器)
- **数据库**: MongoDB (Docker容器)
- **反向代理**: Nginx (Docker容器)
- **容器编排**: Docker Compose

## 环境要求

### 服务器配置
- **操作系统**: Ubuntu 20.04+ / CentOS 8+ / Windows Server 2019+
- **CPU**: 2核心以上
- **内存**: 4GB以上
- **存储**: 50GB以上可用空间
- **网络**: 公网IP地址

### 软件依赖
- Docker 20.10+
- Docker Compose 2.0+
- Git
- 域名和SSL证书 (可选)

## 部署步骤

### 1. 服务器准备

#### 安装Docker和Docker Compose
```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

#### 配置防火墙
```bash
# Ubuntu UFW
sudo ufw allow 22/tcp      # SSH
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS
sudo ufw allow 8080/tcp    # 应用端口
sudo ufw enable

# CentOS firewalld
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

### 2. 代码部署

```bash
# 克隆代码仓库
git clone https://github.com/your-repo/rural-art-platform.git
cd rural-art-platform

# 切换到生产分支
git checkout main
```

### 3. 环境配置

#### 创建生产环境配置
```bash
# 复制环境配置模板
cp backend/.env.example backend/.env
```

#### 配置生产环境变量 (backend/.env)
```env
# 服务器配置
NODE_ENV=production
PORT=3000

# 数据库配置 (生产环境密码)
MONGODB_URI=mongodb://admin:your-strong-password@mongodb:27017/rural-art-platform?authSource=admin

# JWT配置 (生产环境密钥)
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_EXPIRES_IN=7d

# AI服务配置 (生产环境密钥)
GZHU_AGENT_API_URL=https://api.gzhu.edu.cn/agent
GZHU_AGENT_KEY=your-production-gzhu-agent-key
COZE_API_URL=https://api.coze.com/v1
COZE_API_KEY=your-production-coze-api-key
COZE_IMAGE_ENHANCE_BOT_ID=your-production-image-enhance-bot-id
COZE_STYLE_TRANSFER_BOT_ID=your-production-style-transfer-bot-id
COZE_PATTERN_GENERATE_BOT_ID=your-production-pattern-generate-bot-id
COZE_SMART_REPAIR_BOT_ID=your-production-smart-repair-bot-id

# 文件上传配置
UPLOAD_PATH=./uploads
MAX_FILE_SIZE=10485760

# 日志配置
LOG_LEVEL=info
```

#### 更新Docker Compose配置
编辑 `docker-compose.yml` 中的敏感信息：
```yaml
services:
  mongodb:
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: your-strong-password  # 修改为强密码
      MONGO_INITDB_DATABASE: rural-art-platform
```

### 4. 使用部署脚本

#### 一键部署
```bash
# 赋予执行权限
chmod +x deploy.sh

# 构建并启动生产环境
./deploy.sh prod up

# 检查服务状态
./deploy.sh prod status
```

#### 手动部署
```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看服务状态
docker-compose ps
```

### 5. 初始化数据

```bash

# 初始化课程数据
docker-compose exec backend node seed-courses.js

# 验证数据库连接

```

## SSL证书配置 (可选)

### 使用Let's Encrypt免费证书

```bash
# 安装Certbot
sudo apt install certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo crontab -e
# 添加以下行
0 12 * * * /usr/bin/certbot renew --quiet
```

### 更新Nginx配置支持HTTPS
```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    # SSL配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
    ssl_prefer_server_ciphers off;
    
    # 其他配置...
}
```

## 备份策略

### 数据库备份
```bash
# 手动备份
./deploy.sh prod backup

# 自动备份脚本
sudo nano /usr/local/bin/backup-rural-art.sh
```

备份脚本内容：
```bash
#!/bin/bash
BACKUP_DIR="/backup/rural-art/$(date +%Y%m%d_%H%M%S)"
mkdir -p $BACKUP_DIR

# 备份数据库
docker-compose exec -T mongodb mongodump --uri="mongodb://admin:your-password@localhost:27017/rural-art-platform?authSource=admin" --out=/tmp/backup
docker cp $(docker-compose ps -q mongodb):/tmp/backup $BACKUP_DIR/mongodb

# 备份上传文件
docker cp $(docker-compose ps -q backend):/app/uploads $BACKUP_DIR/uploads

# 压缩备份
tar -czf $BACKUP_DIR.tar.gz -C /backup/rural-art $(basename $BACKUP_DIR)
rm -rf $BACKUP_DIR

# 清理旧备份 (保留30天)
find /backup/rural-art -name "*.tar.gz" -mtime +30 -delete
```

### 定时备份
```bash
# 设置定时任务
sudo crontab -e
# 添加以下行 (每天凌晨2点备份)
0 2 * * * /usr/local/bin/backup-rural-art.sh
```

## 故障排除

### 常见问题

1. **容器启动失败**
   ```bash
   # 查看详细错误信息
   docker-compose logs backend
   
   # 检查配置文件
   docker-compose config
   ```

2. **数据库连接失败**
   ```bash
   # 检查MongoDB容器状态
   docker-compose ps mongodb
   
   # 测试数据库连接
   docker-compose exec mongodb mongosh --eval "db.adminCommand('ismaster')"
   ```

3. **前端访问404**
   ```bash
   # 检查Nginx配置
   docker-compose exec nginx nginx -t
   
   # 重新加载Nginx配置
   docker-compose exec nginx nginx -s reload
   ```

4. **API请求失败**
   ```bash
   # 检查后端健康状态
   curl -f http://localhost:8080/api/health
   
   # 查看后端日志
   docker-compose logs -f backend
   ```

### 紧急恢复
```bash
# 停止所有服务
docker-compose down

# 从备份恢复数据库
tar -xzf /backup/rural-art/20240120_020000.tar.gz -C /tmp
docker-compose up -d mongodb
docker cp /tmp/20240120_020000/mongodb $(docker-compose ps -q mongodb):/tmp/restore
docker-compose exec mongodb mongorestore /tmp/restore

# 重启所有服务
docker-compose up -d
```

## 安全建议

### 基础安全
1. **更改默认密码**: 修改MongoDB、JWT等默认密码
2. **定期更新**: 保持系统和Docker镜像更新
3. **防火墙配置**: 只开放必要端口
4. **SSL证书**: 使用HTTPS加密传输
5. **访问控制**: 限制管理员访问权限

### 高级安全
1. **容器安全扫描**: 使用工具扫描镜像漏洞
2. **网络隔离**: 使用Docker网络隔离服务
3. **日志监控**: 监控异常访问和错误日志
4. **备份加密**: 对备份文件进行加密存储

## 扩展部署

### 负载均衡
对于高并发场景，可以部署多个后端实例：

```yaml
# docker-compose.scale.yml
services:
  backend:
    deploy:
      replicas: 3
  
  nginx:
    volumes:
      - ./nginx.loadbalance.conf:/etc/nginx/nginx.conf:ro
```

### 数据库集群
对于高可用需求，可以配置MongoDB副本集：

```yaml
# docker-compose.replica.yml
services:
  mongodb-primary:
    image: mongo:6.0
    command: mongod --replSet rs0 --bind_ip_all
  
  mongodb-secondary:
    image: mongo:6.0
    command: mongod --replSet rs0 --bind_ip_all
```
