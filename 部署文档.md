# 乡村艺术学习平台部署指南 (阿里云+宝塔Linux)

## 概述

本文档提供乡村艺术学习平台在阿里云服务器上使用宝塔Linux面板的完整部署指南，包括Docker容器化部署、宝塔面板管理、SSL证书配置、监控和备份策略。

**🚀 快速部署（推荐）**
如果您希望快速部署系统，请直接跳转到 [简化部署方案](#简化部署方案) 章节。

**宝塔面板专用部署**
针对宝塔面板用户，我们提供了专门的部署脚本，支持直接拉取代码或上传压缩包：

```bash
# Linux 宝塔面板
./deploy-baota.sh

# Windows 宝塔面板
.\deploy-baota.bat
```

**域名配置**
本部署指南已适配域名：`ai-future-rural-art.top`

## 系统架构

### 生产环境架构
```
[用户] → [阿里云SLB/CDN] → [宝塔Nginx:80/443] → [Docker容器组]
                                                    ├── [前端容器:80]
                                                    ├── [后端API:3001] 
                                                    └── [MongoDB:27017]
```

### 技术栈
- **前端**: Vue.js 3 + Vite + Tailwind CSS (Nginx容器)
- **后端**: Node.js 18 + Express (Docker容器)
- **数据库**: MongoDB 6.0 (Docker容器)
- **服务器管理**: 宝塔Linux面板 7.7+
- **云服务**: 阿里云ECS + 对象存储OSS (可选)
- **容器编排**: Docker Compose

## 环境要求

### 阿里云ECS配置
- **实例规格**: ecs.c6.large (2核4G) 或更高
- **操作系统**: CentOS 7.6+ / Ubuntu 20.04+
- **系统盘**: 40GB SSD云盘
- **数据盘**: 100GB SSD云盘 (推荐)
- **网络**: 专有网络VPC + 弹性公网IP
- **安全组**: 开放22, 80, 443, 8888端口

### 宝塔面板要求
- **宝塔Linux面板**: 7.7.0+
- **Docker管理器**: 已安装
- **Nginx**: 1.20+ (宝塔内置)
- **SSL管理器**: 已安装 (Let's Encrypt)
- **文件管理器**: 已安装
- **进程管理器**: 已安装

### 项目端口配置
- **前端开发服务器**: 5713 (本地开发)
- **后端API服务**: 3001 (开发) / 3000 (生产)
- **前端生产容器**: 80 (容器内部)
- **MongoDB**: 27017 (容器内部)
- **宝塔面板**: 8888
- **Nginx代理**: 80/443 (对外访问)

## 宝塔面板专用部署方案

### 概述

针对使用宝塔面板的用户，我们提供了专门的部署脚本，支持两种代码获取方式：
1. **Git拉取**：直接从代码仓库拉取最新代码
2. **压缩包上传**：手动上传项目压缩包到服务器

### 系统要求

- 已安装宝塔面板（Linux或Windows版本）
- Docker 和 Docker Compose（可通过宝塔面板安装）
- Node.js 18+ 和 npm（可通过宝塔面板安装）
- Git（仅Git拉取方式需要）

### 部署步骤

#### 方式一：Git拉取部署

1. **登录服务器**
   ```bash
   # SSH登录到服务器
   ssh root@your-server-ip
   ```

2. **下载部署脚本**
   ```bash
   # 下载Linux版本脚本
   wget https://raw.githubusercontent.com/your-repo/rural-art-platform/main/deploy-baota.sh
   chmod +x deploy-baota.sh
   
   # 或下载Windows版本脚本
   # curl -O https://raw.githubusercontent.com/your-repo/rural-art-platform/main/deploy-baota.bat
   ```

3. **运行部署脚本**
   ```bash
   # Linux环境
   ./deploy-baota.sh
   
   # Windows环境
   # .\deploy-baota.bat
   ```

4. **选择Git拉取方式**
   - 脚本会提示选择代码获取方式，选择 `1) Git拉取`
   - 输入Git仓库地址
   - 输入分支名称（默认：develop）

#### 方式二：压缩包上传部署

1. **准备项目压缩包**
   - 将项目代码打包为 `.zip` 格式
   - 确保压缩包包含完整的项目文件

2. **上传到宝塔面板**
   - 登录宝塔面板
   - 进入文件管理
   - 导航到 `/www/wwwroot/ai-future-rural-art.top/`
   - 上传项目压缩包

3. **运行部署脚本**
   ```bash
   # 下载并运行脚本
   wget https://raw.githubusercontent.com/your-repo/rural-art-platform/main/deploy-baota.sh
   chmod +x deploy-baota.sh
   ./deploy-baota.sh
   ```

4. **选择压缩包方式**
   - 脚本会提示选择代码获取方式，选择 `2) 手动上传压缩包`
   - 脚本会自动检测并解压压缩包

### 部署过程说明

部署脚本会自动执行以下步骤：

1. **环境检查**
   - 检查宝塔面板环境
   - 验证Docker和Node.js安装
   - 检查必要的系统依赖

2. **项目设置**
   - 创建项目目录：`/www/wwwroot/ai-future-rural-art.top/`
   - 备份现有项目（如果存在）
   - 获取项目代码（Git或压缩包）

3. **环境配置**
   - 自动生成 `.env` 和 `backend/.env` 配置文件
   - 配置域名和端口设置
   - 设置数据库连接参数

4. **依赖安装**
   - 安装前端依赖：`npm install`
   - 安装后端依赖：`cd backend && npm install`

5. **项目构建**
   - 构建前端项目：`npm run build`
   - 准备生产环境文件

6. **服务部署**
   - 停止现有Docker服务
   - 构建并启动新的Docker容器
   - 配置Nginx反向代理

7. **服务验证**
   - 等待服务启动完成
   - 检查服务健康状态
   - 配置防火墙规则

### 故障排除

#### 常见问题

1. **Docker服务启动失败**
   ```bash
   # 检查Docker状态
   systemctl status docker
   
   # 启动Docker服务
   systemctl start docker
   ```

2. **端口被占用**
   ```bash
   # 检查端口占用
   netstat -tlnp | grep :80
   netstat -tlnp | grep :3000
   
   # 停止占用端口的进程
   sudo kill -9 <PID>
   ```

3. **权限问题**
   ```bash
   # 修改项目目录权限
   sudo chown -R www:www /www/wwwroot/ai-future-rural-art.top/
   sudo chmod -R 755 /www/wwwroot/ai-future-rural-art.top/
   ```

4. **数据库连接失败**
   ```bash
   # 检查MongoDB容器状态
   docker logs rural-art-mongodb
   
   # 重启MongoDB容器
   docker-compose -f docker-compose.simple.yml restart mongodb
   ```

### 安全建议

1. **修改默认密码**
   - 修改 `backend/.env` 中的 `JWT_SECRET`
   - 修改 `docker-compose.simple.yml` 中的数据库密码

2. **配置SSL证书**
   - 在宝塔面板中为域名配置SSL证书
   - 启用HTTPS访问

3. **防火墙配置**
   - 在宝塔面板中开放必要端口：80, 443
   - 关闭不必要的端口访问

```bash
wget -O deploy-quick.sh https://raw.githubusercontent.com/your-repo/rural-art-platform/main/scripts/deploy-quick.sh
```

### 快速部署脚本内容

创建 `scripts/deploy-quick.sh`：
```bash
#!/bin/bash
# 乡村艺术平台一键部署脚本

set -e

echo "=== 乡村艺术平台一键部署脚本 ==="
echo "开始时间: $(date)"

# 检查系统
if [[ ! -f /etc/redhat-release ]] && [[ ! -f /etc/lsb-release ]]; then
    echo "错误: 仅支持 CentOS 7+ 或 Ubuntu 18+"
    exit 1
fi

# 更新系统
echo "[1/8] 更新系统..."
if command -v yum &> /dev/null; then
    yum update -y
    yum install -y wget curl git
elif command -v apt &> /dev/null; then
    apt update && apt upgrade -y
    apt install -y wget curl git
fi

# 安装宝塔面板
echo "[2/8] 安装宝塔面板..."
if [[ ! -d /www/server/panel ]]; then
    if command -v yum &> /dev/null; then
        wget -O install.sh http://download.bt.cn/install/install_6.0.sh
        bash install.sh ed8484bec
    else
        wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh
        bash install.sh ed8484bec
    fi
else
    echo "宝塔面板已安装，跳过..."
fi

# 安装Docker
echo "[3/8] 安装Docker..."
if ! command -v docker &> /dev/null; then
    curl -fsSL https://get.docker.com -o get-docker.sh
    sh get-docker.sh
    systemctl start docker
    systemctl enable docker
    
    # 安装Docker Compose
    curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
    chmod +x /usr/local/bin/docker-compose
else
    echo "Docker已安装，跳过..."
fi

# 克隆项目代码
echo "[4/8] 获取项目代码..."
cd /www/wwwroot
if [[ ! -d rural-art-platform ]]; then
    git clone https://github.com/your-repo/rural-art-platform.git
else
    cd rural-art-platform
    git pull origin main
    cd ..
fi

cd rural-art-platform

# 配置环境变量
echo "[5/8] 配置环境变量..."
cp .env.example .env
cp backend/.env.example backend/.env

# 生成随机密码
MONGO_PASSWORD=$(openssl rand -base64 32)
JWT_SECRET=$(openssl rand -base64 64)

# 更新配置文件
sed -i "s/your-strong-password/$MONGO_PASSWORD/g" backend/.env
sed -i "s/your-super-secret-jwt-key-change-this-in-production.*/$JWT_SECRET/g" backend/.env
sed -i "s/your-domain.com/ai-future-rural-art.top/g" backend/.env
sed -i "s/your-domain.com/ai-future-rural-art.top/g" .env

# 构建和启动服务
echo "[6/8] 构建和启动服务..."
docker-compose build
docker-compose up -d

# 等待服务启动
echo "[7/8] 等待服务启动..."
sleep 60

# 初始化数据
echo "[8/8] 初始化数据..."
docker-compose exec -T backend node scripts/create-admin.js
docker-compose exec -T backend node scripts/seed-courses.js

# 配置防火墙
echo "配置防火墙..."
if command -v firewall-cmd &> /dev/null; then
    firewall-cmd --permanent --add-port=80/tcp
    firewall-cmd --permanent --add-port=443/tcp
    firewall-cmd --permanent --add-port=8888/tcp
    firewall-cmd --reload
elif command -v ufw &> /dev/null; then
    ufw allow 80/tcp
    ufw allow 443/tcp
    ufw allow 8888/tcp
fi

# 获取服务器IP
SERVER_IP=$(curl -s ifconfig.me)

echo "=== 部署完成 ==="
echo "完成时间: $(date)"
echo ""
echo "访问信息:"
echo "前端地址: http://$SERVER_IP"
echo "宝塔面板: http://$SERVER_IP:8888"
echo "API健康检查: http://$SERVER_IP/api/health"
echo ""
echo "默认管理员账号:"
echo "用户名: admin@example.com"
echo "密码: admin123"
echo ""
echo "重要提醒:"
echo "1. 请及时配置域名解析"
echo "2. 建议在宝塔面板中申请SSL证书"
echo "3. 请修改默认管理员密码"
echo "4. 数据库密码: $MONGO_PASSWORD"
echo "5. JWT密钥已自动生成"
echo ""
echo "如需查看服务状态: docker-compose ps"
echo "如需查看日志: docker-compose logs -f"
```


```nginx
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```

```bash
# - HTTP: 80/TCP, 源：0.0.0.0/0
# - HTTPS: 443/TCP, 源：0.0.0.0/0
# - 宝塔面板: 8888/TCP, 源：您的管理IP
# - 拒绝其他所有端口

# 2. Nginx安全配置
# 在网站配置中添加安全头

add_header X-Content-Type-Options nosniff always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' data:;" always;
```
