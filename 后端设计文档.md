### 1.1 核心技术

- **数据库**: MongoDB (推荐MongoDB Atlas)

- **开发环境**: nodemon (热重载)
- **代码规范**: ESLint + Prettier
- **API文档**: Swagger/OpenAPI
- **测试框架**: Jest
- **日志管理**: Winston

#### Coze智能体调用服务
```javascript
// services/cozeService.js
class CozeService {
  constructor() {
    this.baseURL = process.env.COZE_API_BASE_URL;
    this.apiKey = process.env.COZE_API_KEY;
  }

  async callAgent(agentId, userData, requestData) {
    const payload = {
      agent_id: agentId,
      user_context: userData,
      request_data: requestData,
      session_id: this.generateSessionId()
    };

    const response = await fetch(`${this.baseURL}/agents/${agentId}/invoke`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${this.apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(payload)
    });

    return await response.json();
  }

  // 四个智能体的具体调用方法
  async enhanceImage(userData, imageData, options) {
    return await this.callAgent('enhance-agent-id', userData, { imageData, options });
  }

  async transferStyle(userData, imageData, styleOptions) {
    return await this.callAgent('style-agent-id', userData, { imageData, styleOptions });
  }

  async generatePattern(userData, generationParams) {
    return await this.callAgent('pattern-agent-id', userData, generationParams);
  }

  async repairImage(userData, imageData, repairOptions) {
    return await this.callAgent('repair-agent-id', userData, { imageData, repairOptions });
  }
}
```

### 7.2 Docker部署配置

#### docker-compose.yml
```yaml
version: '3.8'
services:
  # 前端服务
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - app-network

  # 后端服务
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - MONGODB_URI=mongodb://mongo:27017/rural-art
      - JWT_SECRET=${JWT_SECRET}
      - COZE_API_KEY=${COZE_API_KEY}
      - GZHU_AGENT_KEY=689ebf9ac99c2fdc9fea136d1648e507
    depends_on:
      - mongo
    networks:
      - app-network
    volumes:
      - ./uploads:/app/uploads

  # 数据库服务
  mongo:
    image: mongo:6.0
    ports:
      - "27017:27017"
    volumes:
      - mongo-data:/data/db
    networks:
      - app-network
    environment:
      - MONGO_INITDB_DATABASE=rural-art

  # Nginx反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "443:443"
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - frontend
      - backend
    networks:
      - app-network

volumes:
  mongo-data:

networks:
  app-network:
    driver: bridge
```

### 11.2 数据库索引设计

#### 用户集合索引
```javascript
// 用户登录查询
db.users.createIndex({ "email": 1 }, { unique: true })
db.users.createIndex({ "username": 1 }, { unique: true })

// 用户统计查询
db.users.createIndex({ "stats.level": -1 })
db.users.createIndex({ "stats.totalPoints": -1 })
```

#### 动态集合索引
```javascript
// 动态列表查询
db.posts.createIndex({ "createdAt": -1 })
db.posts.createIndex({ "author": 1, "createdAt": -1 })
db.posts.createIndex({ "tags": 1 })

// 热门动态查询
db.posts.createIndex({ "stats.likes": -1, "createdAt": -1 })
```

#### 课程集合索引
```javascript
// 课程分类查询
db.courses.createIndex({ "category": 1, "createdAt": -1 })
db.courses.createIndex({ "difficulty": 1 })

// 热门课程查询
db.courses.createIndex({ "stats.enrollments": -1 })
db.courses.createIndex({ "stats.rating": -1 })
```

## 13. 技术难点和解决方案

### 13.1 高并发处理
**问题**: 用户量增长后的并发访问压力
**解决方案**:
- 使用Redis缓存热点数据
- 数据库连接池优化
- API请求限流
- CDN静态资源分发

### 13.2 数据一致性
**问题**: 积分、关注等数据的一致性保证
**解决方案**:
- MongoDB事务处理
- 乐观锁机制
- 数据校验和修复脚本

### 13.3 AI服务集成
**问题**: 第三方AI服务的稳定性和成本控制
**解决方案**:
- 多个AI服务商备选方案
- 请求失败重试机制
- 用户使用频率限制
- 成本监控和预警

### 13.4 文件存储管理
**问题**: 大量图片和视频文件的存储和访问
**解决方案**:
- 云存储服务(OSS/COS)
- 图片压缩和格式优化
- CDN加速分发
- 定期清理无用文件
