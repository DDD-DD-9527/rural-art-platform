# 课程字段设计文档

## 概述

本文档详细描述了乡村艺术平台课程模块的数据库字段结构，基于当前 MongoDB 数据库中的 Course 模型设计。

## 课程模型 (Course Schema)

### 1. 基本信息字段

| 字段名 | 类型 | 必填 | 限制 | 描述 |
|--------|------|------|------|------|
| `title` | String | ✓ | 最大100字符 | 课程标题 |
| `description` | String | ✓ | 最大1000字符 | 课程描述 |
| `thumbnail` | String | ✓ | - | 课程封面图片URL |
| `creator` | ObjectId | ✓ | 引用User模型 | 课程创作者ID |

### 2. 分类与难度

| 字段名 | 类型 | 必填 | 可选值 | 描述 |
|--------|------|------|--------|------|
| `category` | String | ✓ | 见下方分类列表 | 课程分类 |
| `difficulty` | String | ✓ | beginner, intermediate, advanced | 难度等级 |
| `tags` | Array[String] | - | 每个标签最大20字符 | 课程标签 |

#### 课程分类 (Category) 枚举值

**艺术教育类：**
- `traditional-crafts` - 传统工艺
- `painting` - 绘画艺术
- `sculpture` - 雕塑艺术
- `textile` - 纺织艺术
- `pottery` - 陶艺
- `woodwork` - 木工艺
- `paper-art` - 纸艺
- `folk-art` - 民间艺术
- `calligraphy` - 书法

**数字技能类：**
- `other` - 其他（主要用于数字技能课程）

### 3. 课程内容 (lessons)

课程内容以数组形式存储，每个课时包含以下字段：

| 字段名 | 类型 | 必填 | 限制 | 描述 |
|--------|------|------|------|------|
| `title` | String | ✓ | 最大100字符 | 课时标题 |
| `description` | String | - | 最大500字符 | 课时描述 |
| `videoUrl` | String | - | - | 视频链接 |
| `duration` | Number | - | 最小1分钟 | 课时时长（分钟） |
| `order` | Number | ✓ | - | 课时顺序 |
| `isPreview` | Boolean | - | 默认false | 是否为预览课时 |

#### 课时材料 (materials)

每个课时可包含所需材料列表：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| `name` | String | 材料名称 |
| `description` | String | 材料描述 |
| `optional` | Boolean | 是否可选（默认false） |

#### 课时步骤 (steps)

每个课时可包含详细步骤：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| `stepNumber` | Number | 步骤编号 |
| `title` | String | 步骤标题 |
| `description` | String | 步骤描述 |
| `images` | Array[String] | 步骤图片URL列表 |
| `tips` | String | 操作提示 |

### 4. 统计数据 (stats)

| 字段名 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `enrolledCount` | Number | 0 | 报名人数 |
| `completedCount` | Number | 0 | 完成人数 |
| `viewCount` | Number | 0 | 浏览次数 |
| `likeCount` | Number | 0 | 点赞数 |
| `rating.average` | Number | 0 | 平均评分（0-5） |
| `rating.count` | Number | 0 | 评分人数 |

### 5. 课程设置 (settings)

| 字段名 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `isPublished` | Boolean | false | 是否已发布 |
| `isFree` | Boolean | true | 是否免费 |
| `price` | Number | 0 | 课程价格（最小0） |
| `allowComments` | Boolean | true | 是否允许评论 |
| `allowDownload` | Boolean | false | 是否允许下载 |

### 6. 课程要求 (requirements)

#### 前置条件
| 字段名 | 类型 | 描述 |
|--------|------|------|
| `prerequisites` | Array[String] | 前置课程或技能要求 |

#### 所需材料
| 字段名 | 类型 | 描述 |
|--------|------|------|
| `materials.name` | String | 材料名称 |
| `materials.description` | String | 材料描述 |
| `materials.estimatedCost` | Number | 预估费用 |
| `materials.optional` | Boolean | 是否可选 |

#### 所需工具
| 字段名 | 类型 | 描述 |
|--------|------|------|
| `tools.name` | String | 工具名称 |
| `tools.description` | String | 工具描述 |
| `tools.optional` | Boolean | 是否可选 |

### 7. 学习目标 (learningObjectives)

| 字段名 | 类型 | 限制 | 描述 |
|--------|------|------|------|
| `learningObjectives` | Array[String] | 每项最大200字符 | 学习目标列表 |

### 8. 时间信息

| 字段名 | 类型 | 限制 | 描述 |
|--------|------|------|------|
| `estimatedDuration` | Number | 最小1分钟 | 预估总时长（分钟） |
| `publishedAt` | Date | - | 发布时间 |
| `createdAt` | Date | 自动生成 | 创建时间 |
| `updatedAt` | Date | 自动更新 | 更新时间 |

## 数据库索引

为提高查询性能，在以下字段上建立了索引：

- `creator` - 创作者索引
- `category` - 分类索引
- `difficulty` - 难度索引
- `settings.isPublished` - 发布状态索引
- `stats.rating.average` - 评分索引（降序）
- `stats.enrolledCount` - 报名人数索引（降序）
- `createdAt` - 创建时间索引（降序）
- `publishedAt` - 发布时间索引（降序）
- `tags` - 标签索引
- 全文搜索索引：`title`, `description`, `tags`

## 虚拟字段

### totalDuration
计算所有课时的总时长

### completionRate
计算课程完成率（completedCount / enrolledCount）

## 实例方法

- `publish()` - 发布课程
- `unpublish()` - 取消发布
- `incrementViewCount()` - 增加浏览次数
- `incrementLikeCount()` - 增加点赞数
- `decrementLikeCount()` - 减少点赞数
- `updateRating(newRating, isNewRating)` - 更新评分

## 静态方法

- `getPopularCourses(limit)` - 获取热门课程
- `getCoursesByCategory(category, limit)` - 按分类获取课程
- `searchCourses(query, options)` - 搜索课程

## 前后端功能匹配分析

### 管理员创建课程功能对比

#### 前端表单组件分析

**CourseCreateModal.vue (简化版)**
- 基本信息：标题、描述、分类、难度、时长、价格
- 课程封面：单个图片上传
- 标签管理：动态添加/删除标签
- 课程设置：免费课程、允许评论、立即发布
- 字段映射：无特殊映射，直接提交

**CourseUploadForm.vue (完整版)**
- 基本信息：标题、描述、分类、难度、时长
- 课程封面：图片上传预览
- 课程视频：多个视频文件上传
- 课程材料：PDF/DOC文档上传
- 标签管理：动态标签系统
- 课程设置：草稿/发布状态、免费课程
- 字段映射：`level` → `difficulty`, `thumbnail` → `image`

#### 后端API支持分析

**字段验证规则 (courseRoutes.js)**
```javascript
// 支持的字段
- title: 1-100字符，必填
- description: 1-1000字符，必填
- thumbnail: 必填
- category: 枚举值验证，必填
- difficulty: beginner/intermediate/advanced，必填
- tags: 可选数组，每个标签≤20字符
- lessons: 可选数组
- learningObjectives: 可选数组
- settings.price: 可选，≥0的数字
```

**创建课程API (courseController.js)**
- 直接接收请求体数据
- 自动添加创建者ID
- 使用Course模型保存
- 返回填充创建者信息的课程数据

### 字段匹配情况

#### ✅ 完全匹配的字段
| 前端字段 | 后端字段 | 验证规则 | 状态 |
|---------|---------|---------|------|
| title | title | 1-100字符 | ✅ 匹配 |
| description | description | 1-1000字符 | ✅ 匹配 |
| category | category | 枚举验证 | ✅ 匹配 |
| tags | tags | 数组，≤20字符/项 | ✅ 匹配 |

#### ⚠️ 需要映射的字段
| 前端字段 | 后端字段 | 映射方式 | 状态 |
|---------|---------|---------|------|
| level | difficulty | CourseUploadForm映射 | ⚠️ 部分支持 |
| thumbnail | thumbnail | CourseUploadForm错误映射为image | ❌ 映射错误 |
| duration | estimatedDuration | CourseUploadForm映射 | ⚠️ 字段不匹配 |
| status | settings.isPublished | 发布状态映射 | ⚠️ 结构不匹配 |

#### ❌ 不匹配的功能

**前端支持但后端缺失：**
1. **视频文件上传** - CourseUploadForm支持多视频上传，但后端验证规则中无videos字段
2. **材料文件上传** - CourseUploadForm支持材料上传，但后端验证规则中无materials字段
3. **价格字段** - CourseCreateModal支持价格设置，但CourseUploadForm不支持
4. **课程状态管理** - 前端有draft/published状态，后端通过settings.isPublished管理

**后端支持但前端缺失：**
1. **课时管理** - 后端支持lessons数组，但创建表单中不支持课时编辑
2. **学习目标** - 后端支持learningObjectives，但前端表单中无此字段
3. **课程要求** - 后端模型支持requirements，但前端表单中无此字段

### 关键问题识别

#### 🚨 严重问题
1. **字段映射错误**：CourseUploadForm将thumbnail映射为image，但后端期望thumbnail
2. **分类枚举不匹配**：前端分类选项与后端枚举值可能不一致
3. **文件上传功能缺失**：前端支持视频和材料上传，但后端验证规则不支持

#### ⚠️ 中等问题
1. **字段命名不统一**：duration vs estimatedDuration
2. **状态管理方式不同**：前端status vs 后端settings.isPublished
3. **功能完整性差异**：两个前端表单功能不一致

#### 💡 建议改进
1. **统一字段映射**：修正CourseUploadForm的字段映射逻辑
2. **完善后端验证**：添加videos和materials字段验证
3. **统一前端表单**：整合两个创建表单的功能
4. **完善课时管理**：在创建表单中添加课时编辑功能

## 注意事项

### 分类映射
- 前端显示的分类名称需要与数据库枚举值对应
- 建议在前端维护分类映射表
- **重要**：CourseUploadForm的分类选项与后端枚举值不匹配

### 时间戳
- `createdAt` 和 `updatedAt` 由 Mongoose 自动管理
- `publishedAt` 需要在发布时手动设置

### 数据验证
- 所有必填字段都有相应的验证规则
- 文件上传需要验证格式和大小
- **注意**：前端文件上传功能与后端验证规则不匹配

### 性能优化
- 课程列表查询时排除详细的课时内容
- 使用索引优化常用查询

### 扩展性
- 课程模型支持灵活的课时结构
- 统计数据可以根据需要扩展
- 设置字段支持未来功能扩展
